UX 퀴즈! 나의 UX 상식은 몇 점일까?

 

요약 :
당신의 UX 지식을 테스트 해보자. 모든 질문과 정답은 작년 우리가 발행한 아티클을 기반으로 만들었다.
 

  1. 권위 원칙 (authority principle)은 무엇을 말하는 것인가?

    a. 사람들은 권위 있는 사람들에 반대하려는 경향이 있다.
    b. 사람들은 권위 있는 사람들을 따르려는 경향이 있다.
    c. 만약 권위의 상징이 포함된 사이트라면 사람들은 사용성 문제를 넘어가는 경향이 있다.
    d. 권위있는 사이트는 사이트를 처음 본 0.5초 만에 평가된다.

 

  1. F자형 웹 읽기 패턴 (F-shaped web-reading) 중 옳은 것은?

    a. 웹을 스캔할 때 항상 F자 형태로 스캔하는 것은 아니다.
    b. F자형 패턴은 사용자에게 긍정적이다.
    c. F자형 패턴은 기업에게 긍정적이다.
    d. 디자인으로 F자형 패턴을 방해할 수는 없다.

 

  1. Vortex라는 현상을 설명하는 사용자 행동 패턴 (user-behavior patterns)은?

    a. 여러 페이지를 연속해서 열어서 해당 페이지 항목을 저장하고 나중에 다시 방문하는 것
    b.다른 창이나 탭 사이를 빠르고 자주 전환하는 것
    c. (일련의 무작위의 인터랙션 다음에 따라오는) 디지털 공간으로 몰입하게 해주는 단발성의 인터랙션
    d. 화면 특정 부분을 반복해서 보는 것

 

  1. 역 피라미드 쓰기 스타일(inverted-pyramid writing style)이란?
    a. 아티클에서 가장 중요한 정보 먼저 시작하는 글쓰기 스타일
    b. 학교 논문 같은 것
    c. 인쇄물에는 적합하지만 웹으로는 적합하지 않은 것
    d. 이해력에 부정적인 영향을 주는 것

 

  1. 사용성 테스트 설정 중 subjects study design을 설명하는 것은?
    a. 테스트하는 동안 서로 다른 두 사람이 협력하는 것
    b. 서로 다른 두 사람이 동일한 작업을 같은 시간, 다른 장소에서 작업하는 것
    c. 동일한 참가자에게 다른 버전의 디자인을 모두 테스트하는 것
    d. 각자 다른 사람들에게 디자인을 한 가지 버전 씩 테스트해서 한 사람당 하나의 버전만 볼 수 있도록 하는 것
     

  1. 공감지도란 무엇인가?
    a. 사용자 유형에 대해 알고 있는 정보를 나타내는 공동작업 시각화 (collaborative visualization)
    b. (1) 다양한 서비스 구성요소와 (2) 고객 여정 지도에서 터치 포인트와 관련 있는 프로세스 간의 관계를 보여주는 다이어그램
    c. 특정 비즈니스나 제품과 관련 있는 목표를 달성하기 위해 사람이 겪는 프로세스의 시각화
    d. 목표를 달성하기 위해 ‘일반적인’ 사람이 겪는  end-to-end 경험의 시각화

 

  1. 다음 중 post-task 설문지가 아닌 POST-TEST(효과 측정 테스트)는?
    a. Single Ease Questionnaire
    b. 시스템 유용성 척도 (System Usability Scale (SUS))
    c. NASA-TLX
    d. Subjective Mental Effort Questionnaire (SMEQ)

 

  1. 다음 중 디자인 평가가 아닌 것은?
    a. 휴리스틱 평가(Heuristic evaluation)
    b. 과제 분석 (Task analysis)
    c. 독립 디자인 비평 (Standalone design critique)
    d. UX 전문가 리뷰(UX Expert review)

 

  1. 외부 기억장치(external memory)란?
    a. 현재 작업과 관련된 정보를 저장하는 mental buffer 또는 스크래치 패드
    b. 짧은 시간 동안 단어, 문장, 개념 같은 정보를 저장할 수 있는 뇌 프로세스
    c. 오랜 시간 동안 단어, 문장, 개념 같은 정보를 저장할 수 있는 뇌 프로세스
    d. 사용자가 작업 중에 필요한 정보에 접근하고 저장할 수 있게 도와주는 도구나 UI 기능

 

  1. 다음 중 50가지 항목이 포함된 모바일 캐러셀(carousel)에서 가장 불리한 점은?
    a. 완벽하다는 환상을 만들어 낸다.
    b. 캐러셀 내부에 도달하려면 순차적으로 접근해야 한다.
    c. 캐러셀 내부에 도달하려면 직접 접근해야 한다.
    d. 화면에서 너무 많은 공간이 필요할 것이다.  

 

정답

  1. b / The Authority Principle 아티클 참고

  2. a /  F-Shaped Pattern of Reading on the Web: Misunderstood, But Still Relevant (Even on Mobile) 아티클 참고

  3. c / The Vortex: Why Users Feel Trapped in Their Devices 아티클 참고
    * 해설 :a = page parking; ; b = multitasking; d = exhaustive review

  4. a /  Inverted Pyramid: Writing for Comprehension 아티클 참고

  5. d /  Between-Subjects vs. Within-Subjects Study Design 아티클 참고

  6. a /  Empathy Mapping: The First Step in Design Thinking 아티클 참고

  7. b /  Beyond the NPS: Measuring Perceived Usability with the SUS, NASA-TLX, and the Single Ease Question After Tasks and Usability Tests 아티클 참고

  8. b /  UX Expert Reviews 아티클 참고

  9. d /  Working Memory and External Memory 아티클 참고

  10. b /  Carousels on Mobile Devices 아티클 참고
     

당신의 점수는 몇 점인가요?

10  : 다음에 채용 공고를 내면 이력서를 보내주시길

8-9 : 당신은 UX 전문가. 80%를 맞춘 것은 UX Certification에서 필요한 것 이다. 이 퀴즈는 좋은 연습이 되지만 UX Certification에는 포함되지 않는다. 

6-7 : UX에 대해 잘 알고 있지만, 내년에는 더 열심히 공부하고 싶을 것 같다. 우리의 뉴스레터를 구독해보길

1-5 : 좋은 소식은 지금 당장 더 열심히 공부할 수 있다는 것이다. 각 답변 공개에 제공된 참고 아티클을 모두 읽어보길

0 : 사용자 경험에 대해 관심이 있는가? 공부하고, 참고 아티클을 다시 읽고 퀴즈를 재도전해보자.
