테스크 플로우 VS 유저 플로우
원문 바로보기
학생들이 인터렉션 디자인을 배울 때 공통적으로 묻는 것은 “Task flow와 User flow의 차이점은 무엇인가?” 이다.

모든 디자이너들이 두 가지 개념을 조금 다르게 이해하고 있는 것 같긴 하지만, 두 개념을 구별하는 관점을 한 가지 알려주려고 한다. 

Task flow와 User test 모두 일반적으로 인터렉션 디자인 단계의 UX설계 프로세스에서 유용한 결과물이며, 우리는 이 과정 속에서 디자인의 내용 및 기능에 걸쳐 구조, 계층, 조직 및 관계를 계획한다. 유저들이 제품, 서비스, 경험 디자인을 이용할 때, 원하는 목표를 달성하기 위해서 디자인을 유동적으로 탐색할 수 있도록.

내가 Task flow와 User flow를 구별하는 방법  : Task Flow는 단일 테스크에 초점을 맞추는 반면, User flow는 특정한 유저(타겟 오디언스나 퍼소나) 에 초점을 맞추는 경향이 있다. 

벤 다이어그램은 task flows와 user flows 사이의 차이점과 비슷한 점을 보여준다. 
Task flow 
  • 간단한 단일 경로 simple, single pathways 
  • 갈라지지 않음 No branches 
  • 더 큰 User flow의 부분이 될 수 있음. Can be part of a larger user flow 
  • 사용자의 관점으로 생각할 수 있음. Can be thought of as the user’s perspective 
  • 오직 특정 테스크의 페이지, 스크린, 단계만을 보여줌 Just show the pages/screens/steps of a specific task (not the user actions and decisions)
  • 시작 지점과 끝나는 지점을 드러내지 않음 Don’t show explicit entry (start) and exit (end) point

Task flow & User flow 
  • 사람들이 디자인을 통해 progress/sequence/flow 를 어떻게 보여주는가가 목적. Purpose is to show how people progress/sequence/flow through a design 
  • 설계 중 혹은 마지막에 인터랙션 기자인 이슈를 파악하는 것을 의미 Meant to catch interaction design issues. Such as dead ends or gaps in design.
  • 앱 맵, 사이트 맵과 함께 계획 툴로 사용됨 Used as planning tools in conjunction with app maps and site maps
  • 와이어프레임을 하기 전, 디자인해야 할 주요 페이지, 스크린을 정할 때 사용. Used to determine the key pages/ screens that need to be designed. Prior to starting wireframing

User flow 
  • 복잡한 경로 Complex pathways 
  • 사용자 의사결정 지점/노드 표시 Show user decision points/nodes 
  • 다른 경로로 갈라질 수 있음 Can branch out 
  • 시작 지점과 끝나는 지점을 보여줌 Show entry/start and exit/end points
  • 사용자 인터랙션, 사용자 의사결정, 시스템 결정뿐만 아니라 제안한 페이지/ 스크린을 보여줌 Show proposed pages/screens. Plus user interaction, user Decisions, and system decisions 
  • 특정 유저 / 타겟 / 오디언스 / 퍼소나    Specific to particular users/ target / audiences / personas
  • 다른 사용자에게 다른 방식으로 어떻게 비슷한 목표를 달성하는지 보여줄 때 사용할 수 있음. Can be used to show how different users might accomplish similar goals in different ways
Task Flow는 사람이 특정한 목표나 최종점에 도달하기 위한 하이레벨 단계로 보이는 직선으로 되어있다. Task flow는 의사결정 지점에서 다양한 선택지로 여정을 이어가지 않는 대신에, 순차적이고 일관된 하나의 경로로 단순하게 흐르는 경향이 있다.

펜케이크 레시피를 위한 Task Flow다 

Task flow는 순차적이고 철저하게 디자인되고, 복잡할 수 있는 User flow를 디자인하기 전에 도움을 준다.

반면  User flow는 디자인에 바로 쓸 수 있도록  특정 퍼소나 경로를 보여주는 경향이 있다. 

Task flow가 User flow과정속에 포함될 수 있지만, Task flow와 달리 User flow는 의사 결정 지점이 있다. 설계된 화면 속 요소들과 상호작용 할 때에 퍼소나가 하는 선택에 따라, 원하는 목표를 향한 사용자 여정은 다를 수 있다. User flows는 키 페이지 (웹 사이트일 경우), 혹은 키 스크린 (앱일 경우), user actions, user 결정 포인트, 시스템 결정 같은 여러 유형의 요소가 있다. 플로우를 시작하고 끝내는 여러 가지 방법이 있는 Task flow보다 User flows가 더 복잡하다.

사용자의 다양한 스토리를 나타내고, 제품 및 서비스를 사용하는 다양한 종류의 사람들에게 맞는 적합한 경로를 확인하는 데에 User flow를 설계하는 것이 도움이 될 수 있다.

user flow을 디자인하는 것은, 사용자가 디자인 안에서 오도 가도 못하는 "데드 엔드 (dead end)"를 찾아서 제거하고, 사용자 선택을 설명하고 존중하며, 상호 작용 전반에 걸쳐 명확하고 유동적으로 디자인할 수 있다

user flows와 task flows에 대해 어떻게 생각하는가? 두 가지를 어떻게 구별하는가? user flows에서 나타나는 task flows를 어떻게 설명하는가? 당신의 생각을 알려주시길.
